# Wazuh AI Dashboard Summary

## Purpose

`wazuh-ai-dashboard` is a local HTTP SOC dashboard built with NiceGUI. It displays Wazuh alert and full-log analysis that has already been generated by local AI workers.

The dashboard is read-only for Wazuh and network response:

```text
No Wazuh modification
No active response
No IP blocking
```

It also includes `SOC AI Chat`, which calls the local Ollama model only when the user asks a question. The chat uses the selected alert/log as context and is separate from the realtime AI pipeline.

Telegram integrations are separate services:

```text
wazuh-local-ai.service sends deduplicated alert notifications.
wazuh-telegram-ai-chat.service provides Telegram SOC AI chat.
```

## Data Sources

```text
/var/log/wazuh-local-ai.jsonl
/var/log/wazuh-local-ai-full-log.jsonl
```

The first file is used for alert-focused views. The second file is used for full-log hunting.

## Main Modes

### Demo Mode

Default mode for the current Kali-to-Win11 demo.

It only shows:

```text
100503 - Port 135 probe detected from Kali to Win11
100504 - Port 139 probe detected from Kali to Win11
100505 - Port 445 probe detected from Kali to Win11
```

Pipeline:

```text
Kali nmap -> Win11 Firewall DROP -> Wazuh Alert -> Local AI Triage -> SOC Dashboard
```

### SOC Mode

Shows all alert AI records from:

```text
/var/log/wazuh-local-ai.jsonl
```

Records are grouped by source type:

```text
Windows
Linux
OPNsense
Wazuh
Unknown
```

### Full Log Mode

Shows observer records from:

```text
/var/log/wazuh-local-ai-full-log.jsonl
```

This mode is for hunting and context review. It is not the main alert queue.

## SOC AI Chat

The chat panel supports demo-friendly analysis prompts:

```text
Explain Alert
Next Steps
Report Conclusion
Compromise?
Hunt Related
```

It is useful for showing interactive local AI analysis without sending data to an external service.

## Event Dedupe

The dashboard reads JSONL records that may contain both `pending` and final `ok` entries for the same `event_id`.

Current behavior:

```text
If pending and ok exist for the same event_id, show ok.
```

This keeps the demo panel focused on the final analysis while preserving complete JSONL history on disk.

## Telegram

Telegram notification:

- sent by `wazuh-local-ai.service`
- only for `status=ok`
- only for configured demo rules and minimum risk
- burst dedup TTL: 60 seconds

Telegram SOC chat:

- service: `wazuh-telegram-ai-chat.service`
- supports `/latest`
- supports quick buttons similar to dashboard SOC AI Chat
- uses Local AI and JSONL context
